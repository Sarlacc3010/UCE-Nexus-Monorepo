# MS-11: Catalog Service

A high-performance catalog microservice written in **Rust** that exposes a **gRPC** interface. It manages university infrastructure records (such as laboratories, equipment, locations, and capacity) for the **UCE-Nexus** Smart-Campus ecosystem.

## 🏗️ Architecture Role

This service manages the static and structural information of campus infrastructure:
1. **Infrastructure Directory**: Provides query details on classroom locations, capacity, and active status.
2. **Database Seeding**: Automatically checks, creates, and seeds default university lab data (Cisco lab, computation rooms, auditoriums) on startup if PostgreSQL is empty.
3. **Availability Queries**: Answers gRPC availability checks from other services (such as the AI agent or scheduling engines).

---

## 🛠️ Tech Stack

- **Language**: Rust (2021 Edition)
- **Asynchronous Runtime**: `tokio`
- **gRPC Framework**: `tonic` (HTTP/2)
- **Database Engine**: `sqlx` (Asynchronous PostgreSQL driver)
- **Serialization**: `serde` and `serde_json`
- **Reflection**: `tonic-reflection` for development testing

---

## 🔌 gRPC Services

The service compiles proto contracts via `build.rs` using `tonic-build` and exposes the interface defined in `packages/proto-contracts/catalog.proto`:

### `CatalogService`
* **`GetLaboratory(GetLaboratoryRequest) returns (LaboratoryResponse)`**
  * Queries details for a specific laboratory by its unique string code. Returns name, location, capacity, equipment array, and active status.
* **`CheckAvailability(CheckAvailabilityRequest) returns (AvailabilityResponse)`**
  * Checks if a specific laboratory is active and available for booking on a specific date.

---

## 💾 Database Schema

The service connects to a PostgreSQL instance (`postgres-catalog` database) and maintains a single table:

- **`laboratories`**:
  - `id`: UUID (Primary Key, autogenerated)
  - `code`: VARCHAR(50) (Unique code, e.g. `LAB-Cisco-01`)
  - `name`: VARCHAR(100) (Descriptive name)
  - `location`: VARCHAR(150) (Building and room coordinate)
  - `capacity`: INT (Maximum capacity of students)
  - `equipment`: TEXT[] (Array of equipment available, e.g. Routers, PCs)
  - `is_active`: BOOLEAN (Operational status)

## 🚀 How to Run & Test

### Run Locally (Development)

1. Make sure you have the Rust toolchain installed.
2. Ensure database is running and access credentials are correct.
3. Build and run the project:
   ```bash
   cargo run
   ```

### Run Tests
```bash
cargo test
```

### Test gRPC via CLI (`grpcurl`)
If you have `grpcurl` installed, you can query the reflection endpoint:
```bash
grpcurl -plaintext localhost:50052 list
grpcurl -plaintext -d '{"code": "LAB-Cisco-01"}' localhost:50052 catalog.CatalogService/GetLaboratory
```

### Run with Docker
```bash
docker build -t nexus-ms-11-catalog .
```
